# Query panel in brain-map — design

2026-07-06. Approved via brainstorming.

## Goal

Interactive panel inside `brain-map.html` to query the indexed content — the `node brain.js "question"` experience in the browser, anchored to the graph.

## Decisions

- **Mode:** Both — ranked candidate list + full `retrieve()` answer for the top (or clicked) candidate.
- **Location:** inside `brain-map.html`, extending the existing `#panel`.
- **Graph link:** yes — clicking a candidate zooms/highlights its node in the graph.
- **Engine:** server-side (approach A). One `GET /query` endpoint reusing the exact CLI `retrieve()`. No client-side scoring duplication. Serve-mode-only, same constraint as attach/detach.

## Server (brain.js)

- `retrieve(question, opts)` gains structured output alongside the existing text:
  - `candidates`: up to 3 of `{name, layer, path, desc, score}`
  - `answer`: `{path, head, body, pointer}` (pointer = followed-pointer target path, if any)
  - CLI text output unchanged.
- `serve()` gains `GET /query?q=<question>` → JSON of the above.
  - Optional `&pick=<path>`: force the answer section from that candidate's file (for clicks on candidate #2/#3). `pick` must be a path present in the index (same guard as `/file`).

## Panel (brain-map.html, generated by buildMap)

- New "ask the brain" input in `#panel`. The existing `#q` node-filter input keeps its job — different purposes, different boxes.
- Enter → fetch `/query` → render:
  - up to 3 candidate chips (name + score)
  - answer section: `📍 path § head` + body, scrollable results pane
- Click a chip → refetch with `pick`, re-render answer, zoom/highlight that node in the graph.
- Opened from `file://` → show the same "needs serve mode" hint pattern `#fbhint` uses.

## Error handling

- No match → same "no index match" message the CLI prints.
- `/query` with missing/empty `q` → 400 JSON error, matching existing endpoint style.

## Testing

- One case added to `node brain.js test`: call `retrieve()` with structured output and assert JSON shape (candidates array, answer body non-empty for a known query). No server spin-up needed.
